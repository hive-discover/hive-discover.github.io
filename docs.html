Hive Discover - API Documentation

Because we are better at backend work and not good designers, we want to leave the frontend to you! We provide the API and you present our results nicely - deal?!

Important Notes

You can integrate Hive Discover into your service if you accept and comply with the following rules:

- You are only asking information from yourself / a user who wants this

- For functions realised with Hive Discover, you refer to @action-chain. A simple "Powered by @action-chain" is sufficient

V1 Routes

All requests must be sent to "https://api.hive-discover.tech/v1". There is no HTTP port for security reasons.

GET - Ping - "/"

Just ask if the service is currently active and ready to work. Should return a JSON response with the status "ok".

Accounts

     GET - Information - "/accounts"

Get information about an account registered with Hive Discover. If this account has not accepted the Privacy Policy, it will respond with a status of "failed". If the user has no posts/votes in the last 100 days, empty categories and languages will be displayed. In this example request, the "ac-bot" account is queried. 
Parameters as query variables:

Username
HiveSigner Access Token

     GET - Feed - "/accounts/feed"

This is one of the main features of Hive Discover: the personalised feed. If the user has not accepted our privacy policy, a "failed" status will be displayed. If the user has not made any Votes/Posts in the last 100 days, an empty feed is returned. 
Parameters as query variables:

Username
Amount
HiveSigner Access Token

Searching

     POST - Search for Posts - "/search/posts"

Search by topics or categories and get matching posts from the last 100 days. You can sort by score/oldest/latest or by personalised sorting, which requires the username and access token. If the account does not agreed to our Privacy Policy or has no posts/votes in the last 100 days, this will fail!
Parameters in body as a json-object:

query
Amount
ScoreLatestOldestPersonalized

Necessary points for the sorting mode "personalised":

Username
HiveSigner Access Token

     POST - Search Similar Posts - "/search/similar-post"

Search for similar contributions. This compares based on our AI results and returns the closest posts using Knn Search. If the post has not been written in the last 100 days / has NSFW content and does not contain English elements, it will be answered with a status "failed".
Parameters in body as a json-object:

Author
Permlink
Amount

     POST - Search for Accounts - "/search/accounts"

Search by topics and get matching posts from the last 100 days. You can sort by score/oldest/latest or by personalised sorting, which requires the username and access token. If the account does not agreed to our Privacy Policy or has no posts/votes in the last 100 days, this will fail!
Parameters in body as a json-object:

query
Amount

     POST - Search Similar Accounts - "/search/similar-account"

Search for similar contributors on HIVE. Based on the results of our AI, an account profile is created, which is then compared using Knn Search. The closest profiles are then returned. If the user does not exist or has no votes/posts in the last 100 days, a status of "failed" is returned.
Parameters in body as a json-object:

Author
Amount

     POST - Search by Categories - "/search/category"

Search categories based on posts from the last few days. Therefore, all posts can be voted. 46 individual categories can be selected.
